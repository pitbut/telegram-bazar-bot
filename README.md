# Список на базар — Telegram-бот

Жена собирает список в мини-приложении внутри Telegram → мужу приходит уведомление →
он отмечает покупки → жене приходит итог (план/факт/перерасход).

## 1. Создать бота через @BotFather

1. Открой Telegram, найди **@BotFather**, напиши `/start`.
2. Отправь `/newbot`.
3. Придумай имя (например, «Список на базар») — это отображаемое имя.
4. Придумай username бота, обязательно заканчивается на `bot` (например `bazar_pit_bot`).
5. BotFather пришлёт **токен** вида `123456789:AAExampleTokenXXXXXXXXXXXXXXXXXXXXX`.
   Сохрани его — это секретный ключ, его никому не показывай и не выкладывай в открытый код.

## 2. Задеплоить backend на Render.com

1. Зарегистрируйся на render.com (можно через GitHub).
2. Залей эту папку в свой репозиторий на GitHub (или подключи Render к пустому репо и залей файлы).
3. На Render: **New → Web Service** → выбери репозиторий.
4. Настройки:
   - **Runtime**: Python 3
   - **Build command**: `pip install -r requirements.txt`
   - **Start command**: `gunicorn app:app`
5. В разделе **Environment** добавь переменные:
   - `BOT_TOKEN` = токен из BotFather
   - `APP_URL` = `https://<имя-твоего-сервиса>.onrender.com` (Render покажет этот адрес после первого деплоя — впиши его сюда и передеплой)
6. Дождись деплоя. Render даст тебе адрес вида `https://bazar-pit.onrender.com`.

> Бесплатный план Render "засыпает" после 15 минут простоя и просыпается ~30 секунд при первом обращении — для семейного use-case это не критично, но если раздражает, можно взять платный instance ($7/мес) чтобы не засыпал.

## 3. Подключить вебхук

Выполни один раз (замени токен и адрес):

```bash
curl "https://api.telegram.org/bot<ТВОЙ_ТОКЕН>/setWebhook?url=https://<твой-сервис>.onrender.com/webhook"
```

Должно прийти `{"ok":true,"result":true,...}`.

## 4. Проверка

1. В Telegram найди своего бота по username, нажми `/start`.
2. Выбери «Я жена» с одного аккаунта/телефона, «Я муж» — с другого.
3. С аккаунта жены: `/list` → откроется мини-приложение → добавь товары → «📨 Отправить мужу».
4. Мужу придёт обычное уведомление Telegram с кнопкой «Открыть список».
5. Муж отмечает покупки → «Завершить поход» → жене придёт итог.

## Что можно улучшить дальше (не критично для старта)

- **Проверка подлинности** запросов из Mini App (Telegram передаёт `initData`, которую можно
  проверить HMAC-подписью токеном бота — сейчас это не проверяется, для семейного использования
  не критично, но если планируешь показывать это чужим людям, стоит добавить).
- Сейчас один "открытый список" на всю систему — если нужно несколько активных списков
  одновременно (например, у разных пар), потребуется привязать trips к паре chat_id.
- История походов уже пишется в базу (`/api/history`), но в интерфейсе Mini App вкладки
  истории пока нет — легко добавить по образцу того, что было в обычном HTML-варианте.
